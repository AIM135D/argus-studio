# Synthetic Demo

ARGUS Studio includes a self-contained demo so every major workflow can be evaluated without private data or external downloads.

## Contents

- 48 synthetic 640×384 industrial scenes;
- YOLO labels for `person`, `helmet`, and `vest`;
- an approximately six-second MJPG AVI walkthrough;
- two Ultralytics-style experiment directories with curves and declared edge metrics;
- prediction JSON containing TP, FP, FN, and low-confidence examples;
- a sample audit narrative.

All images and video frames are generated by `core/argus_core/demo.py`. The scenes are geometric illustrations, not photographs of real people or facilities.

## No model weights

The experiment records contain `results.csv`, `args.yaml`, and `edge_metrics.json`. `model_size_mb` is a declared synthetic comparison metric. The repository intentionally contains no `.pt`, `.pth`, `.onnx`, or engine file.

## Walkthrough

1. Start ARGUS Studio and select **加载内置 Demo**.
2. Open **数据审计** and run the YOLO checks.
3. Open **标签复核** to browse boxes and add a review note.
4. Open **训练实验** and import the two Demo runs.
5. Open **模型对比** to compare balanced, accuracy, and deployment strategies.
6. Open **推理回放** to inspect precomputed detection outcomes.
7. Open **边缘交付** to generate a configuration-only RDK X5/Linux handoff.

## Regenerate

```bash
.venv/bin/python scripts/generate_demo.py
```

Tests create isolated Demo directories under temporary paths. Committed prediction and dataset paths are relative so the public repository contains no contributor-specific filesystem information.
